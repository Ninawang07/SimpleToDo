# 极简桌面 To-Do 产品规划

## 1. 产品定位

| 维度 | 定义 |
|------|------|
| **目标用户** | 个人开发者/效率控，对内存敏感 |
| **核心价值** | 秒开、常驻不卡、用完即走 |
| **内存目标** | 常驻内存 < 50MB，冷启动 < 500ms |
| **形态** | 桌面常驻小窗口 / 托盘图标一键唤起 |
| **哲学** | 不做项目管理、不做协作、只做清单 |

---

## 2. 功能规格

### 2.1 核心功能（V1.0 必须）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 新增任务 | 输入框回车即添加，支持 `Enter` 快速保存 | P0 |
| 完成任务 | 点击复选框标记完成，已完成任务可折叠 | P0 |
| 编辑任务 | 双击文本进入编辑，失焦自动保存 | P0 |
| 删除任务 | 悬停显示删除按钮 / 左滑删除 | P0 |
| 子任务 | 每个主任务下可展开添加子任务（最多3层） | P0 |
| 任务排序 | 拖拽排序 + 按创建时间/优先级排序切换 | P1 |
| 本地存储 | 数据存本地 JSON/SQLite，无需联网 | P0 |

### 2.2 扩展功能（V1.1 可选）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 优先级标记 | 高/中/低三档，带颜色标识 | P1 |
| 截止日期 | 可选日期，到期前提醒 | P1 |
| 回收站 | 删除后保留7天可恢复 | P2 |
| 导入/导出 | JSON/CSV 格式备份 | P2 |
| 全局快捷键 | `Ctrl+Shift+T` 快速唤起窗口 | P1 |
| 开机自启 | 最小化到托盘启动 | P2 |

### 2.3 明确不做（克制清单）

- 多用户协作 / 云同步
- 标签系统 / 项目管理
- 番茄钟 / 时间追踪
- 富文本编辑 / 附件上传
- 主题皮肤市场

## 3. 交互设计

> **原型图**：`todo-app-prototype.html`（可用浏览器打开查看）

### 3.1 窗口形态（最终版）

```
┌─────────────────────────────────┐
│  ☐ To-Do          ─ □ ✕       │  ← 标题栏（可拖拽移动）
├─────────────────────────────────┤
│  [＋ 添加新任务...          ]    │  ← 输入框
├─────────────────────────────────┤
│ ▼ ☐ 完成主页设计               │  ← 主任务（已展开）
│     ☐ 确定配色方案   5/16 5/18 │  ← 子任务（缩进 + 日期标签右对齐）
│     ☐ 画原型图       5/16      │
│   ☐ 买咖啡豆         5/10      │  ← 无 DDL，标签列不塌陷
│   ☐ 提交季度报告   5/12 逾期   │  ← DDL 逾期显示红色
├─────────────────────────────────┤
│  3 待办 · 1 逾期               │  ← 底部统计
└─────────────────────────────────┘
```

**关键设计决策（最终版）：**
1. 右侧标签区固定 `82px`，两列均分（`flex: 1`），所有行严格对齐
2. 子任务通过 `padding-left: 20px` 缩进，不影响右侧标签列位置
3. 无 DDL 时第二列用空 `span` 占位（visibility: hidden），列宽不塌陷
4. 逾期 DDL 用红色背景 `#E24B4A` + 粉红底色 `#FCEBEB`

### 3.2 交互规则

| 操作 | 响应 |
|------|------|
| `Enter` | 输入框聚焦时添加任务 |
| `Tab` | 在任务间切换焦点 |
| `Delete` | 焦点在任务上时删除 |
| `Ctrl+Shift+T` | 全局唤起/隐藏窗口 |
| 双击任务文本 | 进入编辑模式 |
| 点击 ▼/▶ | 展开/收起子任务 |
| 拖拽任务 | 调整排序 |
| 点击托盘图标 | 显示/隐藏窗口 |
| 关闭按钮 | 最小化到托盘（不退出） |

## 4. 技术选型对比

| 方案 | 代表框架 | 内存占用 | 开发效率 | 分发难度 | 推荐度 |
|------|----------|----------|----------|----------|--------|
| **原生 Win32** | Win32 API / WPF | ~15-30MB | 低 | 低 | ⭐⭐⭐ |
| **现代原生** | WinUI 3 / Windows App SDK | ~25-40MB | 中 | 低 | ⭐⭐⭐⭐ |
| **轻量跨平台** | Tauri (Rust+WebView2) | ~35-60MB | 高 | 低 | ⭐⭐⭐⭐⭐ |
| **传统跨平台** | Electron | ~100-200MB | 高 | 低 | ⭐⭐ |

### 推荐方案：Tauri

**理由：**
1. **内存友好**：Rust 后端 + 系统 WebView2，无内置 Chromium，比 Electron 轻 60%+
2. **开发高效**：前端用 React/Vue + Tailwind，开发体验接近 Web
3. **原生能力**：支持全局快捷键、系统托盘、本地文件读写
4. **单文件分发**：可打包为单个 `.exe`，无运行时依赖
5. **安全默认**：Rust 内存安全，权限模型清晰

**架构图：**

```
┌─────────────────────────────────────────┐
│              Frontend (WebView2)         │
│         React + Tailwind CSS             │
│   组件：TaskList / TaskItem / SubTask    │
└──────────────┬──────────────────────────┘
               │  Tauri IPC (invoke)
┌──────────────▼──────────────────────────┐
│              Backend (Rust)              │
│    数据层：SQLite (rusqlite)             │
│    系统层：快捷键、托盘、窗口管理         │
└─────────────────────────────────────────┘
```

## 5. 数据模型

```typescript
interface Task {
  id: string;           // UUID
  content: string;      // 任务内容
  completed: boolean;   // 是否完成
  priority: 'high' | 'medium' | 'low';
  dueDate?: string;     // ISO 日期
  parentId?: string;    // 父任务ID（子任务用）
  order: number;        // 排序权重
  createdAt: string;    // 创建时间
  updatedAt: string;    // 更新时间
}
```
